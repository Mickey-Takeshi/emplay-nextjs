# ブログコンテンツ AI検出回避チェックリスト

ブログ記事や説明文を作成・レビューする際に、以下の項目を確認してください。
AI生成コンテンツと判定されるリスクを下げるためのガイドラインです。

---

## 1. 構文パターンの多様化

- [ ] **冒頭文が毎回同じパターンになっていないか**
  - NG例: 毎記事「〇〇カテゴリでは、△△を発信しています。」
  - OK例: 疑問形、体験談、データの引用など、記事ごとに変える
- [ ] **締めの文が画一的でないか**
  - NG例: 全記事「〜をお伝えします」「〜を解説します」で終わる
  - OK例: 問いかけ、具体的なアクション提案、読者への語りかけなど
- [ ] **見出し構成が毎回同じテンプレートになっていないか**
  - NG例: 毎回「概要→メリット→手順→まとめ」の4段構成
  - OK例: テーマに合わせて構成を変える

## 2. 箇条書き・リストの扱い

- [ ] **箇条書きの項目数が毎回同じでないか**（例: 必ず7つ）
- [ ] **箇条書きのフォーマットが完全に統一されていないか**
  - NG例: 全て「**太字キーワード**: カンマ区切りの説明」
  - OK例: 短い文、長めの補足、数字データなど混在させる
- [ ] **箇条書きだけで構成していないか**
  - 文章の中に自然に情報を織り込む方が人間らしい

## 3. 語彙と表現の自然さ

- [ ] **同じ接続詞を多用していないか**
  - 「また」「さらに」「加えて」の連発はAI臭が強い
- [ ] **不自然に丁寧すぎる表現がないか**
  - 「〜することが重要です」「〜が不可欠です」の乱用を避ける
- [ ] **具体性のない修飾語を使っていないか**
  - 「包括的な」「革新的な」「最適な」「効果的な」→ 具体的な内容に置き換える
- [ ] **カタカナ語・ビジネス用語を詰め込みすぎていないか**
  - 自然な日本語とのバランスを取る

## 4. 人間らしさの演出

- [ ] **読者への語りかけや問いかけが含まれているか**
  - 「〜ではないでしょうか」「〜という経験はありませんか？」
- [ ] **書き手の視点や意見が入っているか**
  - 「個人的には」「弊社の経験では」「正直なところ」
- [ ] **具体的なエピソードや数字があるか**
  - 抽象的な説明だけでなく、「たとえば〜」「実際に〜」
- [ ] **あえて完璧でない表現を残しているか**
  - 口語的な表現、カジュアルなトーンを適度に混ぜる
  - 「ぶっちゃけ」「ぶっちゃけ」まではいかなくとも「正直」「実は」など

## 5. 構造レベルでの注意

- [ ] **複数コンテンツ間でテンプレート感がないか**
  - 同時期に作った複数記事を並べて比較する
  - 文字数、段落数、見出し数がほぼ同じなら要注意
- [ ] **導入 → 本文 → まとめ の流れが機械的でないか**
  - 本文途中に余談や補足を挟む
  - 「ここまで読んだ方は」など途中で読者に声をかける
- [ ] **全体の文字数が不自然に均一でないか**
  - 記事の長さにはテーマに応じた自然なバラつきがあるべき

## 6. SEOとの両立

- [ ] **キーワードを不自然に詰め込んでいないか**
  - 検索キーワードを意識しつつも、自然な文脈で使う
- [ ] **メタディスクリプション（excerpt）が定型文になっていないか**
  - 記事ごとに内容を反映した個別のサマリにする
- [ ] **見出しタグ（h2, h3）にキーワードを詰め込みすぎていないか**

## 7. レビュー時の最終チェック

- [ ] **記事全体を音読して違和感がないか**
  - 声に出して読むと機械的な文章はすぐにわかる
- [ ] **AI検出ツールでスコアを確認したか**
  - GPTZero、Originality.ai などで事前チェック
  - スコアが高い場合は該当箇所を書き直す
- [ ] **第三者（社内メンバーなど）にレビューしてもらったか**

## よくあるNG → OK 変換例

| NG（AI感が強い） | OK（人間らしい） |
|---|---|
| 〇〇について解説します | 〇〇で困っていませんか？ |
| 以下のポイントが重要です | 押さえておきたいのはこの3つ |
| 〜することが可能です | 〜できます |
| 〜を実現します | 〜につながります / 〜が期待できます |
| まず〇〇、次に△△、最後に□□ | 〇〇から始めて、慣れてきたら△△も |
| 包括的なソリューション | （具体的に何をするか書く） |
| それでは見ていきましょう | （削除。すぐ本文に入る） |

---

## 運用ルール

1. **新規記事作成時**: このチェックリストの各項目を確認してから公開する
2. **既存記事の改善時**: AI検出ツールでスコアが高い記事から優先的に改善する
3. **チェックリストの更新**: AI検出技術は進化するため、半年に1回は項目を見直す
