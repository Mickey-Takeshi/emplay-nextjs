## 2. フェーズ別の実施内容

### Phase A: 監査と基盤修正(7/3)
5系統の並列監査(デザイン/SEO-AIO/フォーム/git履歴/競合5社)を実施し、P0から順に修正。
- **お問い合わせフォーム**: E2E実測で動作確認(DB保存→Gmail通知まで)。エラーを「送信完了」と偽装する危険なハンドリングを撤去
- **SEO/AIO基盤**: OG画像(1200×630)・apple-touch-icon新規作成、sitemap自動生成化(ビルド時にDBから生成)、robots.txt正常化(noindexとDisallowの競合解消)、構造化データ(Organization完全住所/FAQPage/BreadcrumbList自動出力)、プライバシーポリシー新設
- **法令・信頼**: フォームにプライバシー同意文言、Careersにハニーポット
- コミット: `afc9aa4`(移植)、`8b3db06`(ヒーロー写真)

### Phase B: コンテンツ第1波(7/4〜7/15)
- 既存118記事のAIっぽさレビュー → **55本リライト適用**(`9a706e3`)
- **新規50本**(SEOキーワード設計)+ **FDE・最新AI 20本** 公開(`9a706e3`, `cba28f2`)

### Phase C: 問い合わせ運用の仕組み化(7/9〜7/10)
- **自動仕分け**: Edge Function `send-contact-email` にルールベース分類器を実装。件名タグ(【要対応の可能性】/【要確認】/【営業の可能性】)+本文バナー+DB保存(classification/lead_score/reasons)。過去分も遡及付与
- **誤判定チューニング**: 「候補日提示」「営業ピッチ語」を営業シグナル化、曖昧語(作りたい等)は単独でlead判定しない(Citrus App誤判定のFB反映)
- **Gmail自動仕分け**: 見込み=スター+「要対応」、営業=受信トレイスキップ+「営業・売り込み」のフィルタ設定済み
- 現在の累計分類: sales 78 / review 22 / lead 1(eclore — 返信下書き作成済み)

### Phase D: CVR/SEO改善第2波(7/10)
